## 표면적 이해
CI/CD 파이프라인 설계 패턴은 소웨어 개발 과정에서 자동화된 빌드, 테스트, 배포를 가능하게 해주는 구조이다.

## Deep Dive

### 왜 필요한가?
- 자동화된 빌드와 테스트를 통해 소프트웨어 개발의 효율성을 높이고 오류를 줄이며 배포 시간을 단축할 수 있다.
- 이전에는 수동으로 빌드와 테스트를 수행하여 시간을 소비하고human 에러가 많이 발생했다.

### 내부 동작 원리
- CI/CD 파이프라인은 Git과 같은 버전 관리 시스템에서 소스 코드를 가져와서 자동으로 빌드, 테스트, 배포하는 과정을 수행한다.
- 빌드 단계에서 소스 코드를 컴파일하여 실행 가능한 파일로 변환한다.
- 테스트 단계에서 자동화된 테스트를 수행하여 코드의 안정성을 확인한다.
- 배포 단계에서 최종적으로 테스트된 결과물을 프로덕션 환경에 배포한다.

```
+---------------+
|  버전 관리  |
+---------------+
       |
       |
       v
+---------------+
|  빌드(BUILD)  |
+---------------+
       |
       |
       v
+---------------+
|  자동 테스트  |
+---------------+
       |
       |
       v
+---------------+
|  배포(DEPLOY)  |
+---------------+
       |
       |
       v
+---------------+
|  프로덕션    |
+---------------+
```

### 코드로 이해하기

```typescript
// Jenkinsfile에 정의된 파이프라인 단계
pipeline {
    agent any
    stages {
        stage('BUILD') {
            steps {
                sh 'make build'
            }
        }
        stage('TEST') {
            steps {
                sh 'make test'
            }
        }
        stage('DEPLOY') {
            steps {
                sh 'make deploy'
            }
        }
    }
}
```

```typescript
// 잘못된 사용 예: 파이프라인마다 중복된 코드를 사용하는 경우
pipeline {
    agent any
    stages {
        stage('BUILD') {
            steps {
                sh 'make clean; make build'
            }
        }
    }
}
pipeline {
    agent any
    stages {
        stage('BUILD') {
            steps {
                sh 'make clean; make build'
            }
        }
    }
}

// 올바른 사용 예: 파이프라인 라이브러리를 사용하여 코드를 재사용하는 경우
def build() {
    sh 'make clean; make build'
}
pipeline {
    agent any
    stages {
        stage('BUILD') {
            steps {
                build()
            }
        }
    }
}
```

### 실전 팁
- Best Practice: 코드 검토를 통해 코드 품질을 향상시킨다.
- 흔한 실수와 해결법: 파이프라인이 실패하고 재시도했을 때, 이전 단계의 결과를 재사용하여 시간을 절약할 수 있다.
- 성능 관련 주의사항: 대용량의 빌드와 배포를 수행할 때에는 성능에 유의하여 적절한 자원 할당이 필요하다.

### 한 줄 정리
CI/CD 파이프라인 설계 패턴은 자동화된 소프트웨어 개발과 배포를 가능하게 함으로써 소프트웨어의 품질과 속도를 높여준다.
