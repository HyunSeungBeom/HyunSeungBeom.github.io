## 표면적 이해
Next.js에서 제공하는 SSR, SSG, ISR, PPR 렌더링 전략을 비교하고 분석하는 포스트입니다.

## Deep Dive

### 왜 필요한가?
- 이러한 렌더링 전략은 사용자에게 빠르고 효율적인 웹 페이지를 제공하는 문제를 해결합니다.
- 이전 방식의 한계는 서버에서 전체 페이지를 렌더링하거나 클라이언트에서만 렌더링하는 것에 대한 한계로, 사용자 요청에 따라 동적으로 콘텐츠를 업데이트하고 캐싱을 효과적으로 활용할 수 없는 문제가 있었습니다.

### 내부 동작 원리
- Next.js의 SSR(Server-Side Rendering)은 사용자 요청시마다 서버에서 페이지를 렌더링하여 응답합니다. SSG(Static Site Generation)는 빌드 시에 페이지를 렌더링하여 정적 파일로 생성합니다. ISR(Incremental Static Regeneration)은 정적 파일을 생성하고 캐싱하는 과정에서 부분적으로 업데이트할 수 있는 기능입니다. PPR(Performance-Optimized Pre-Rendering)은 Next.js 13에서 소개된 프리렌더링 기술로, 더 빠르며 효율적인 렌더링을 제공합니다.
```
                      +---------------+
                      |  사용자 요청  |
                      +---------------+
                             |
                             |
                             v
                      +---------------+
                      |  라우터 처리  |
                      +---------------+
                             |
                             |
                             v
                      +---------------+
                      |  SSR/SSG/ISR  |
                      |  렌더링 전략  |
                      +---------------+
                             |
                             |
                             v
                      +---------------+
                      |  페이지 렌더링  |
                      +---------------+
                             |
                             |
                             v
                      +---------------+
                      |  캐싱 및 응답  |
                      +---------------+
```

### 코드로 이해하기
```typescript
// pages/_app.tsx
import { AppComponent } from 'next/app';

function MyApp({ Component, pageProps }) {
  return <Component {...pageProps} />;
}

export default MyApp;
```

```typescript
// pages/index.tsx
import { GetStaticProps } from 'next';

export const getStaticProps: GetStaticProps = async () => {
  // 정적 파일 생성 시뮬레이션
  const data = await fetch('https://api.example.com/data');
  return {
    props: {
      data: data.json(),
    },
  };
};

function HomePage({ data }) {
  return <div>{data}</div>;
}

export default HomePage;
```

### 비교 분석

| 구분 | SSR | SSG | ISR | PPR |
|------|---|---|---|---|
| 렌더링 시점 | 요청시 | 빌드시 | 빌드시 및 요청시 | 빌드시 및 요청시 |
| 캐싱 여부 | O | O | O | O |
| 콘텐츠 업데이트시 | 전체 페이지 재렌더링 | 부분 업데이트나 전체 페이지 재빌드 | 부분 업데이트나 전체 페이지 재빌드 | 부분 업데이트나 전체 페이지 재빌드 |

### 실전 팁
- 캐싱을 적극적으로 사용하여 성능을하세요. however, 캐싱을 사용할 때 콘텐츠가 업데이트 되는 경우를 고려하여 캐싱을 갱신할 수 있도록 주의하세요.
- Next.js 버전을 latest 버전으로 유지하여 새로운 기능을 활용하세요.
- SSG와 ISR을 함께 사용하여 정적 파일을 생성하고 캐싱하는 과정에서 부분적으로 업데이트할 수 있도록 활용하세요.

### 한 줄 정리
Next.js에서 제공하는 렌더링 전략은 SSR, SSG, ISR, PPR로, 각 렌더링 전략은 렌더링 시점, 캐싱 여부, 콘텐츠 업데이트 시의 동작이 다르며, 이러한 렌더링 전략을 적절하게 활용하여 사용자에게 빠르고 효율적인 웹 페이지를 제공할 수 있습니다.
