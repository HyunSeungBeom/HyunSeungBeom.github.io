## 표면적 이해
Next.js 서버 액션과 데이터 페칭 패턴은 서버 렌더링과 데이터 로딩을 더 효율적으로 관리하는 방법입니다.

## Deep Dive

### 왜 필요한가?
- 이전 방식에서는 페이지 이동이나 데이터 로딩 시마다 전체 페이지를 다시 렌더링해야 하거나, 데이터를 불러와야 하는 문제가 있었습니다. 이러한 방식은 사용자 경험을 좋게 하지 못하고, 서버에 부담을 줄 수 있습니다.
- Next.js 서버 액션과 데이터 페칭 패턴은 이런 문제를 해결하기 위해 필요한 기술입니다. 이 기술을 통해 개발자는 효율적으로 데이터를 불러와서 서버에서 렌더링할 수 있습니다.

### 내부 동작 원리
- Next.js는 서버에서 페이지를 렌더링하고, 클라이언트에서 hydration을 시킵니다. 하지만, 이 방식은 페이지 이동이나 데이터 로딩 시마다 서버에서 페이지를 다시 렌더링해야 하거나, 데이터를 불러와야 하는 문제가 있었습니다.
- Next.js 서버 액션과 데이터 페칭 패턴은 서버에서 페이지를 렌더링하고, 클라이언트에서 추가적인 데이터를 로딩하는 방식을 제공합니다.하면 서버에서 페이지를 렌더링하고, 클라이언트에서 필요한 데이터만 로딩할 수 있습니다.
```
                        +---------------+
                        |  클라이언트   |
                        +---------------+
                             |
                             |  요청
                             v
                        +---------------+
                        |  Next.js 서버 |
                        +---------------+
                             |
                             |  페이지 렌더링
                             v
                        +---------------+
                        |  페이지 데이터  |
                        +---------------+
                             |
                             |  데이터 로딩
                             v
                        +---------------+
                        |  추가 데이터   |
                        +---------------+
```

### 코드로 이해하기
```typescript
// pages/index.tsx
import { GetServerSideProps } from 'next';

export const getServerSideProps: GetServerSideProps = async () => {
  const data = await fetch('https://api.example.com/data');
  const jsonData = await data.json();

  return {
    props: {
      data: jsonData,
    },
  };
};

export default function Home({ data }) {
  return (
    <div>
      <h1>홈페이지</h1>
      <ul>
        {data.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

```typescript
// 잘못된 사용 예
// 직접 데이터를 클라이언트에서 로딩하는 경우
export default function Home() {
  const [data, setData] = useState([]);

  useEffect(() => {
    fetch('https://api.example.com/data')
      .then((data) => data.json())
      .then((jsonData) => setData(jsonData));
  }, []);

  return (
    <div>
      <h1>홈페이지</h1>
      <ul>
        {data.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}

// 올바른 사용 예
// 서버에서 페이지를 렌더링하고, 클라이언트에서 추가적인 데이터를 로딩하는 경우
export const getServerSideProps: GetServerSideProps = async () => {
  const data = await fetch('https://api.example.com/data');
  const jsonData = await data.json();

  return {
    props: {
      data: jsonData,
    },
  };
};

export default function Home({ data }) {
  const [additionalData, setAdditionalData] = useState([]);

  useEffect(() => {
    fetch('https://api.example.com/additional-data')
      .then((data) => data.json())
      .then((jsonData) => setAdditionalData(jsonData));
  }, []);

  return (
    <div>
      <h1>홈페이지</h1>
      <ul>
        {data.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
      <ul>
        {additionalData.map((item) => (
          <li key={item.id}>{item.name}</li>
        ))}
      </ul>
    </div>
  );
}
```

|------|---|---|---|
| 렌더링 | 클라이언트에서 렌더링 | 서버에서 렌더링 | 서버에서 렌더링, 클라이언트에서 추가 데이터 로딩 |
| 데이터 로딩 | 클라이언트에서 데이터 로딩 | 서버에서 데이터 로딩 | 서버에서 데이터 로딩, 클라이언트에서 추가 데이터 로딩 |
| 성능 | 클라이언트에서 데이터 로딩으로 인한 성능 저하 | 서버에서 데이터 로딩으로 인한 성능 향상 | 서버에서 데이터 로딩과 클라이언트에서 추가 데이터 로딩으로 성능 향상 |

### 실전 팁
- Next.js 서버 액션과 데이터 페칭 패턴을 사용하면 서버에서 페이지를 렌더링하고, 클라이언트에서 추가적인 데이터를 로딩할 수 있습니다.하면 사용자 경험을 좋게 할 수 있습니다.
- 하지만, 서버에서 데이터를 로딩하는 경우에는 서버의 부담이 커질 수 있습니다. 따라서, 데이터의 크기와 형식, 그리고 서버의 성능을 고려하여 사용해야 합니다.
- 또한, 클라이언트에서 데이터를 로딩하는 경우에는 데이터의 보안을 고려해야 합니다. 데이터를 암호화하여 전송하고, 클라이언트에서 데이터를 저장하는 경우에는 데이터의 보안을 고려해야 합니다.

### 한 줄 정리
Next.js 서버 액션과 데이터 페칭 패턴은 서버에서 페이지를 렌더링하고, 클라이언트에서 추가적인 데이터를 로딩하는 방식을 제공하여 사용자 경험을 좋게 할 수 있습니다.
