## 표면적 이해
배포 전략은 소프트웨어 배포에 관련된 전략을 의미하며, Blue-Green, Canary, Rolling 배포가 대표적입니다.

## Deep Dive

### 왜 필요한가?
- 이러한 배포 전략이 해결하는 문제는 소프트웨어 배포 시 발생할 수 있는 장애나 오류를 최소화하고, 서비스의 가용성을 높이는 것입니다. 이전 방식의 한계는 배포 시 서비스 중단이나 장애로 인한 사용자 불편을解决하지 못했습니다.

### 내부 동작 원리
- Blue-Green 배포는 두 개의 동일한 환경을 준비하여, 하나의 환경에서 소프트웨어를 배포하고, 테스트한 뒤에 트래픽을 이전하는 방식입니다.
- Canary 배포는 새로운 버전의 소프트웨어를 일부 사용자에게만 배포하여, 문제가 없는지 테스트하는 방식입니다.
- Rolling 배포는 서비스의 일부분을 하나씩 업데이트하여, 서비스 중단을 최소화하는 방식입니다.

```
          +---------------+
          |  LOAD BALANCER  |
          +---------------+
                  |
                  |
                  v
+---------------+---------------+
|         BLUE          |         GREEN         |
|  (OLD VERSION)  |  (NEW VERSION)  |
+---------------+---------------+
                  |
                  |
                  v
          +---------------+
          |  USERS         |
          +---------------+
```

### 코드로 이해하기

```typescript
// 배포 전략을 결정하는 함수
function deployStrategy(strategy: string) {
  if (strategy === 'blue-green') {
    // Blue-Green 배포 로직
  } else if (strategy === 'canary') {
    // Canary 배포 로직
  } else if (strategy === 'rolling') {
    // Rolling 배포 로직
  }
}

// 잘못된 사용 예 (❌)
// deployStrategy('undefined');

// 올바른 사용 예 (✅)
deployStrategy('blue-green');
```

### 비교 분석

| 구분 | Blue-Green | Canary | Rolling |
|------|---|---|---|
| 장애 복구 | 빠름 | 보통 | 느림 |
| 서비스 중단 | 발생 | 최소화 | 최소화 |
| 테스트难도 | 보통 | 어려움 | 보통 |

### 실전 팁
- Best Practice: Blue-Green 배포는 서비스 중단을 최소화하고, 빠른 장애 복구가 필요한 경우에 적합합니다.
- 흔한 실수와 해결법: Canary 배포 시 일부 사용자에게만 배포하는 것을 잊지 않도록 주의해야 합니다.
- 성능 관련 주의사항: Rolling 배포 시 서비스의 일부분을 업데이트하므로, 서비스의 성능에 영향을 주지 않도록 주의해야 합니다.

### 한 줄 정리
배포 전략은 소프트웨어 배포 시 발생할 수 있는 장애나 오류를 최소화하고, 서비스의 가용성을 높이는 데 중요한 역할을 합니다.
